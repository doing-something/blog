## Forward Pass: 뉴런이 답을 내놓는 과정

### 뉴런이란

숫자 여러 개를 받아서 하나의 숫자를 출력하는 계산 단위다.
생물학적 신경세포가 자극을 받아 다음 뉴런으로 신호를 보내는 구조에서 영감을 받아 만들어졌다.

### 가중합: 특징 × 가중치, 다 더하기

뉴런은 입력(특징)을 받으면 각각에 가중치(weight)를 곱한 뒤 전부 더한다(가중합). 여기에 편향(bias)까지 더한 값, 곧이어 활성화 함수에 들어갈 이 값을 z라고 부른다. z라는 글자 자체에 뜻은 없고, 뒤에서 계속 가리키기 위한 이름이다.

```
z = (특징1 × 가중치1) + (특징2 × 가중치2) + ... + 편향
```

### 가중합이 그냥 합과 다른 점

그냥 합은 모든 값을 똑같이 취급해서 더한다. 가중합은 각 값에 중요도(가중치)를 곱한 뒤 더한다.

```
그냥 합:  80 + 90 + 70 = 240
가중합:   (80 × 0.2) + (90 × 0.5) + (70 × 0.3) = 16 + 45 + 21 = 82
```

### 뉴런에서의 가중합

```
특징:   [0.5, 1.0]
가중치: [0.7, -0.4]
편향:   -0.1

z = (0.5 × 0.7) + (1.0 × -0.4) + (-0.1)
  = 0.35 + (-0.4) + (-0.1)
  = -0.15
```

가중치는 각 특징의 중요도다. 크면 그 특징이 결과에 더 큰 영향을 주고, 음수면 반대 방향으로 영향을 준다.

편향은 특징과 무관하게 결과를 전체적으로 위아래로 밀어주는 조정값이다. 양수면 "예" 쪽으로, 음수면 "아니오" 쪽으로 기울어진다.

```
특징1 ──× 가중치1 ──┐
                     ├── 다 더하기 ──+ 편향 ──→ z
특징2 ──× 가중치2 ──┘
```

### 시그모이드: z를 확률로 변환

z는 아무 숫자나 될 수 있다. 그런데 "사과일 확률"처럼 0과 1 사이 값이 필요하므로 시그모이드(sigmoid) 함수를 씌운다.

시그모이드의 출력은 항상 0~1 사이이므로, 보통 "1일 확률"처럼 해석한다.

```
σ(z) = 1 / (1 + e^(-z))
```

e는 약 2.718인 수학 상수다. 시그모이드의 출력은 다음과 같이 결정된다.

- z가 큰 양수 → 1에 가까움
- z가 0 → 정확히 0.5
- z가 큰 음수 → 0에 가까움

S자 곡선 형태로 부드럽게 변한다. 값이 갑자기 끊기지 않고 부드럽게 변하는 점이 나중에 학습(미분)할 때 중요해진다.

```
특징들 → 가중합 + 편향 → z → 시그모이드 → 확률(0~1)
[0.5, 1.0]  (가중치 적용)  -0.15  (S자 곡선)   0.4626
```

이렇게 뉴런의 출력을 0~1 사이로 변환해주는 함수를 활성화 함수(activation function)라고 부른다.

### 손실 함수: MSE

이진 분류에서는 실제로는 MSE보다 Binary Cross-Entropy(BCE)를 더 많이 사용한다.
여기서는 Backpropagation 과정을 단순하게 설명하기 위해 MSE를 사용한다.

뉴런이 0.8이라고 예측했는데 정답이 1이면, 0.2만큼 틀린 것이다. 이 "얼마나 틀렸나"를 숫자로 나타내는 것이 손실(loss)이다.

```
MSE = 평균( (예측 - 정답)² )
```

이름을 거꾸로 읽으면 계산 순서가 된다.

1. Error — 예측에서 정답을 뺀다 (오차)
2. Squared — 제곱한다 (음수 제거 + 큰 오차 강조)
3. Mean — 전부 더해서 개수로 나눈다 (평균)

```
예측: [0.8, 0.3], 정답: [1, 0]
차이: (0.8-1)² + (0.3-0)² = 0.04 + 0.09 = 0.13
평균: 0.13 ÷ 2 = 0.065
```

MSE가 0에 가까울수록 예측이 좋다는 뜻이다. 학습의 목표는 MSE를 줄이는 것이다.

### Forward Pass 전체 흐름

```
입력 특징들 → [가중치 곱하고 더하기 + 편향] → z
                    ↓
              시그모이드 함수
                    ↓
             0~1 사이 값 (보통 확률처럼 해석)
                    ↓
             정답과 비교 → MSE (손실)
```

## Backpropagation: 뉴런을 학습시키는 과정

Forward Pass로 예측을 내놓고 MSE로 점수를 매겼다. 이제 그 점수를 개선하려면 가중치와 편향을 조정해야 한다.
그 과정은 크게 두 단계다.
- Backpropagation: 기울기(gradient)를 계산
- Gradient Descent: 그 기울기를 이용해 가중치를 수정

### 미분: 변화의 속도를 재는 도구

딥러닝에서는 "값을 조금 바꿨을 때 결과가 얼마나 변하는가"를 미분으로 계산한다.
그래프로 보면 기울기이고, 여러 변수에 대한 기울기들을 모아놓은 것이 그래디언트(gradient)다.

알아둘 규칙 하나: 제곱을 미분하면 2가 앞으로 나온다.
```
x²을 미분하면 → 2x
```

MSE 안에 제곱이 있으므로, MSE를 미분하면 2가 등장한다.

### 경사하강법: 손실을 줄이는 방향 찾기

MSE를 줄이려면 가중치를 어느 방향으로 얼마나 바꿔야 하는지 알아야 한다. 그것을 알려주는 것이 그래디언트다.

| 그래디언트 | 가중치를 올리면? | 조치 |
|------------|------------------|------|
| 양수 (+) | MSE 커짐 | 가중치를 내린다 |
| 음수 (−) | MSE 작아짐 | 가중치를 올린다 |

항상 그래디언트의 반대 방향으로 가중치를 바꾸면 MSE가 줄어든다.

```
새 가중치 = 기존 가중치 − 학습률 × 그래디언트
```

빼기(−)를 쓰는 이유: 그래디언트는 MSE가 커지는 방향을 가리키므로, 반대로 가야 MSE가 줄어든다. 이것이 경사 하강(gradient descent)이다.

학습률(learning rate)은 한 번에 얼마나 크게 조정할지 정하는 값이다. 너무 크면 최적점을 넘어가고, 너무 작으면 수렴이 느리다.

### Backpropagation: 기울기를 거꾸로 계산

가중치의 그래디언트를 한 번에 구할 수 없다. 손실은 여러 단계를 거쳐 나왔기 때문이다.

```
w, b → z → y_pred → 손실
```

그래서 거꾸로 따라가며 한 단계씩 미분을 곱해나간다.

#### 1단계: 손실 → y_pred (error)

```
error = y_pred - labels
```

예측과 정답의 차이(오차)다.
MSE를 미분하면 이 오차에 비례하는 형태가 나온다.

#### 2단계: y_pred → z (sigmoid_grad)

```
sigmoid_grad = y_pred * (1 - y_pred)
```

시그모이드의 미분이다. "z를 살짝 바꾸면 y_pred가 얼마나 변하는가"의 답이다.
시그모이드 곡선은 위치에 따라 기울기가 다르다.

```
y_pred = 0.5  → 0.5 × 0.5  = 0.25   (가장 가파름, 잘 변함)
y_pred = 0.9  → 0.9 × 0.1  = 0.09
y_pred = 0.99 → 0.99 × 0.01 = 0.0099 (거의 평평, 잘 안 변함)
```

0.5 근처에서는 조금만 바꿔도 출력이 크게 변하고, 0이나 1에 가까우면 바꿔도 거의 변하지 않는다.

그래서 시그모이드 출력이 0이나 1에 가까워지면 기울기가 매우 작아진다.
깊은 신경망에서는 이런 작은 기울기가 계속 곱해지면서 학습이 잘 안 되는 문제가 생길 수 있다. 이것이 기울기 소실(vanishing gradient)이다.

#### z에 대한 기울기

```
error_grad = (2/n) * error * sigmoid_grad
```

1단계와 2단계를 곱하면 "z를 살짝 바꾸면 손실이 얼마나 변하는가"가 된다. 2/n은 MSE 미분에서 나오는 상수다 (제곱 미분의 2, 평균의 1/n).

```
손실 ← y_pred ← z ← w  (z까지는 같고, 마지막 한 걸음만 다름)
                  ← b
```

#### 가중치 기울기

```
grad_w = features.T @ error_grad
```

z = features @ weights + b 이므로, z를 weights에 대해 미분하면 features가 나온다.
따라서 가중치 기울기는 error_grad에 특징값(features)을 곱한 형태가 된다.

왜 전치(features.T)가 필요한가:
데이터가 3개, 특징이 2개일 때, 원하는 결과는 특징마다 기울기 하나씩 총 2개다.
features는 보통 (데이터 개수 × 특징 개수) 형태다.
전치(features.T)를 하면 (특징 개수 × 데이터 개수)가 되어, 행렬 곱 한 번으로 각 특징의 기울기를 동시에 계산할 수 있다.

#### 편향 기울기

```
grad_b = error_grad.sum()
```

z를 b로 미분하면 1이다. 따라서 편향 기울기는 error_grad을 모든 데이터에 대해 합산한 것이다.

### 오차 신호: error × sigmoid_grad

error_grad의 핵심인 error × sigmoid_grad를 풀어서 보면

```
오차 신호 = (예측 - 정답) × 예측 × (1 - 예측)
            └─────┬─────┘   └──────┬───────┘
          얼마나 틀렸나       바꾸기 쉬운 정도
```

많이 틀렸고 바꾸기도 쉬우면 신호가 강하고(크게 조정), 조금 틀렸거나 바꾸기 어려우면 신호가 약하다(살짝 조정).

```
예측 = 0.7, 정답 = 1

(0.7 - 1) = -0.3               ← 부족함, 올려야 함
0.7 × (1 - 0.7) = 0.21         ← 시그모이드 기울기
오차 신호 = -0.3 × 0.21 = -0.063
```

음수 그래디언트이므로, 가중치가 예측을 높이는 방향으로 조정된다.
실제 업데이트 공식에서 빼기(−)를 쓰므로, 음수 그래디언트를 빼면 가중치가 커지는 방향으로 움직여 예측이 올라간다.

### 배치와 epoch

- 풀배치(full-batch): 모든 데이터를 한꺼번에 보고 평균 기울기를 구한 뒤, 한 번 업데이트한다.
- epoch: 전체 데이터를 한 번 다 보는 것. epochs = 100이면 이 과정을 100번 반복한다.

### 학습의 전체 흐름

각 epoch마다 네 가지를 수행한다.

```
┌──────────── 에폭 1회 ────────────┐
│                                   │
│  ① Forward Pass: 모든 데이터로 예측   │
│       z = features @ weights + b  │
│       y_pred = sigmoid(z)         │
│                                   │
│  ② MSE 기록 (업데이트 전에 기록)  │
│       mse = mean((y_pred-y)²)     │
│                                   │
│  ③ Backpropagation: 기울기 계산     │
│       error, sigmoid_grad         │
│       → error_grad                │
│       → grad_w, grad_b            │
│                                   │
│  ④ 업데이트: 가중치, 편향 수정    │
│       w = w - lr × grad_w         │
│       b = b - lr × grad_b         │
│                                   │
│  → 다음 에폭으로                  │
└───────────────────────────────────┘
        이것을 epochs번 반복
```

## Multi-class Cross-Entropy Loss

### 컴퓨터는 확신 대신 확률을 출력한다

컴퓨터에게 사진을 보여주고 "이게 고양이야? 강아지야? 새야?" 하고 물어보면, 모델은 보통 이렇게 답한다.
"고양이일 확률 70%, 강아지일 확률 20%, 새일 확률 10%"
이런 값을 예측 확률이라고 한다. 전부 더하면 항상 100%가 된다.
그런데 진짜 정답은 고양이인데, 모델이 고양이라고 얼마나 자신 있게 말했는지를 보고 싶다.
만약 고양이 99%는 매우 잘 맞춤이고, 고양이 5%이 매우 틀림이라면, 이 차이를 숫자로 표현할 필요가 있다.

이 "얼마나 틀렸는가"를 하나의 숫자로 나타낸 것이 바로 손실(loss)이다.

### 정답 표현: 원-핫 인코딩

정답도 숫자로 표현해야 한다. 정답 클래스 위치만 1, 나머지는 0으로 채우는 방식이 원-핫 인코딩(one-hot encoding)이다.

```
              고양이  강아지  새
정답    →   [  1,     0,    0 ]
```

### 손실을 재는 척도: 교차 엔트로피

교차 엔트로피(cross-entropy)는 예측 확률과 정답을 비교해서, 예측이 정답에 얼마나 낮은 확률을 줬는지 측정한다.

- 정답 클래스에 90%를 줬다면 정답과 잘 맞음: 교차 엔트로피 낮음
- 정답 클래스에 5%만 줬다면 정답과 크게 어긋남: 교차 엔트로피 높음

이걸 숫자로 계산하려면 로그(log)가 필요하다.

### 로그: 곱셈을 거꾸로 푸는 도구

#### 지수부터 시작

같은 수를 여러 번 곱하는 걸 짧게 쓰는 방법이 지수다.

```
10 × 10 × 10 = 10³
```

- 위에 붙은 3이 지수(exponent): 곱한 횟수
- 아래 10은 밑(base): 무엇을 곱했는지

#### 로그는 지수의 역연산

로그는 결과를 보고 "몇 번 곱했는지"를 역으로 구한다.

1000을 만들려면 10을 몇 번 곱해야 하는가? → 3번 → log₁₀(1000) = 3

#### 로그의 성질

##### 1. 큰 수를 작게 만든다.

10을 10번 곱하면 100억, 20번 곱하면 1000경이다. 로그를 씌우면 각각 10, 20이 된다. 감당하기 어려운 큰 숫자를 다루기 쉬운 크기로 줄여준다.

##### 2. 곱셈을 덧셈으로 바꾼다.

```
log(100 × 1000) = log(100) + log(1000) = 2 + 3 = 5
```

이 성질은 손실 계산에서 중요하게 쓰인다.

#### 자연로그와 e

머신러닝에서 쓰는 로그는 밑이 10이 아니라 e ≈ 2.718...인 자연로그(ln)다. torch.log(), np.log() 모두 자연로그다.
e를 사용하면 미분 계산이 깔끔해져서 수학과 머신러닝에서 널리 사용된다.

#### 확률에 로그를 씌우면: 틀린 정도가 드러난다.

확률은 0에서 1 사이 값이다. 이 범위에서 자연로그를 씌우면:

```
확률 0.9  → ln(0.9)  ≈ -0.1    거의 안 틀림
확률 0.5  → ln(0.5)  ≈ -0.69   어느 정도 틀림
확률 0.1  → ln(0.1)  ≈ -2.30   많이 틀림
확률 0.01 → ln(0.01) ≈ -4.60   크게 틀림
```

확률이 낮을수록 로그 값이 더 큰 음수가 된다.
부호를 뒤집으면(-ln) 이 값이 곧 손실이 된다.

```
손실 = -ln(확률)

손실

 ^

 |\__

 |   \____

 |        \________

 +--------------------> 정답 확률

 0         0.5      1
```

- 정답 확률이 1에 가까우면 손실은 거의 0
- 정답 확률이 낮아질수록 손실이 급격히 커진다
- 틀린 답을 강하게 확신할수록 큰 벌점을 받는다

확률이 0에 다가가면 로그는 마이너스 무한대로 발산한다.
그래서 실제 구현에서는 log(0)이 발생하지 않도록 아주 작은 값(epsilon)을 더하거나, 수치적으로 안정적인 함수를 사용한다.

#### 왜 확률을 그대로 안 쓰고 로그를 씌우는가

##### 1. 낮은 확률에 큰 벌점을 줄 수 있다.

`1 - 확률`을 벌점으로 쓰면, 정답에 0%를 줘도 벌점이 1이고 1%를 줘도 0.99다. 차이가 미미하다.
로그를 쓰면 0에 가까울수록 벌점이 급격히 커져서, 크게 틀린 예측에 강한 페널티를 줄 수 있다.

##### 2. 곱셈을 덧셈으로 바꿔 수치적으로 안전하다.

여러 샘플의 확률을 곱하면 값이 극도로 작아져서 부동소수점 연산에서 문제가 생긴다.
로그를 씌우면 곱셈이 덧셈이 되어 안정적으로 계산할 수 있다.

### 교차 엔트로피 계산 과정

예측과 정답이 다음과 같다고 하자.

```
예측 확률   = [0.7,  0.2,  0.1]    ← 고양이 70%, 강아지 20%, 새 10%
정답(원-핫) = [1,    0,    0  ]    ← 정답은 고양이
```

#### 1단계: 로그 씌우기

```
[0.7,    0.2,    0.1  ]
→ [ln(0.7), ln(0.2), ln(0.1)]
= [-0.36,  -1.61,  -2.30]
```

#### 2단계: 정답과 곱하기 (정답 클래스만 남기기)

```
[1, 0, 0]  ×  [-0.36, -1.61, -2.30]
= [-0.36, 0, 0]
```

실제 딥러닝 프레임워크에서는 메모리 효율 때문에 원-핫 벡터 대신 정답 인덱스만 저장하는 경우가 많다.

#### 3단계: 합산

```
-0.36 + 0 + 0 = -0.36
```

#### 4단계: 부호 반전

```
-(-0.36) = 0.36
```

확률의 로그는 음수이므로, 손실을 양수로 만들기 위해 앞에 마이너스를 붙인다.

이 샘플의 손실 = 0.36

정답 클래스 하나만 쓸 건데 왜 전체에 로그를 씌울까?
수식으로는 정답 클래스의 로그 확률만 필요하지만, 실제 구현에서는 벡터 전체를 한 번에 처리하는 방식이 GPU 연산에 잘 맞는다.
또한 softmax와 로그를 따로 계산하지 않고, 수치적으로 안정적인 log_softmax 형태로 한꺼번에 계산하는 경우가 많다.

### 배치(batch): 여러 샘플을 한꺼번에

실제로는 샘플 한 개가 아니라 수십~수백 개를 묶어서 한꺼번에 처리한다.
이 묶음을 배치(batch)라고 한다.

위 계산을 각 샘플에 대해 수행한 뒤, 전체의 평균을 내면 그것이 평균 교차 엔트로피 손실이다.

## Binary Cross-Entropy Loss

### 이진 분류: 확률 하나만 있으면 된다.

멀티클래스에서는 클래스마다 확률이 하나씩 있었다 ([0.7, 0.2, 0.1]). 이진 분류에서는 "예(1)일 확률" 하나만 있으면 된다.

```
정답:  1       ← 실제로 스팸이었음
예측:  0.9     ← 모델이 출력한 "예일 확률" 90%
```

"예일 확률"이 0.9이면, "아니오일 확률"은 자동으로 1 - 0.9 = 0.1이다.
별도로 출력할 필요가 없다.

### 정답에 따라 보는 확률이 달라진다.

이 부분이 멀티클래스와 다른 점이다. 정답이 1인지 0인지에 따라 어느 확률을 기준으로 손실을 계산할지가 갈린다.

#### 정답이 1(예)일 때 → "예일 확률"이 높았어야 한다.

```
예측 0.9 → "예"라고 할 확률이 높음 → 잘 맞음 → 손실 작음
예측 0.1 → "예"라고 할 확률이 낮음 → 크게 틀림 → 손실 큼
```

#### 정답이 0(아니오)일 때 → "예일 확률"이 낮았어야 한다.

즉, "예일 확률" p는 낮고, "아니오일 확률" 1-p는 높아야 한다.

```
예측 0.1 → "예"라고 할 확률이 낮음 → 아니오를 잘 맞음 → 손실 작음
예측 0.9 → "예"라고 할 확률이 높음 → 아니오를 크게 틀림 → 손실 큼
```

### 공식: 두 갈래를 하나로

```
손실 = -[ y·ln(p) + (1-y)·ln(1-p) ]
```

| 기호 | 뜻 | 코드 |
|------|-----|------|
| `p` | 모델이 출력한 "예(1)일 확률" | `y_pred` |
| `1-p` | 자동으로 정해지는 "아니오(0)일 확률" | `1 - y_pred` |
| `y` | 실제 정답 (0 또는 1) | `y_true` |
| `1-y` | 정답이 아닌 쪽 | `1 - y_true` |

p(예측)와 y(정답)는 다른 값이다. p는 0~1 사이 아무 값이 될 수 있고, y는 0 아니면 1 딱 둘 중 하나다.

### 정답에 따라 항이 하나씩 사라진다.

#### 정답 y = 1일 때

```
-[ 1·ln(p) + 0·ln(1-p) ] = -ln(p)
```

(1-y) 쪽이 0이 되어 사라진다. "예일 확률" p의 로그만 남는다.

#### 정답 y = 0일 때

```
-[ 0·ln(p) + 1·ln(1-p) ] = -ln(1-p)
```

y 쪽이 0이 되어 사라진다. "아니오일 확률" 1-p의 로그만 남는다.

결국 정답 클래스에 해당하는 확률의 로그만 살아남는 구조다. 멀티클래스 교차 엔트로피에서 원-핫 벡터가 정답 위치만 남기는 것과 같은 원리다.

### 멀티클래스와의 관계

이진 교차 엔트로피는 멀티클래스 교차 엔트로피의 클래스가 2개인 특수한 경우다.
멀티클래스에서 클래스가 [예, 아니오] 두 개이고 원-핫 정답이 [1, 0]이면, 예측 확률 [p, 1-p]에 대해 교차 엔트로피를 구하는 것과 동일하다. 다만 클래스가 2개뿐이라 확률 하나로 충분하므로, 별도의 공식으로 쓰는 것이다.

## Root Mean Square Error (RMSE)

### 회귀(Regression)란

앞서 다룬 교차 엔트로피는 분류 문제의 손실 함수였다. 분류는 "어느 클래스인가"를 맞히는 것이다.
회귀는 특정 클래스가 아니라 연속적인 숫자 값을 예측하는 문제다.

- 내일 기온은 몇 도? → 23.5도
- 이 집 값은 얼마? → 4억 2천만 원
- 이 사람 키는? → 175cm

이처럼 연속적인 값을 예측하므로, "얼마나 틀렸나"를 재는 방법도 단순하다. 예측이 정답에서 얼마나 멀리 떨어졌는지를 보면 된다.

### RMSE: 이름을 거꾸로 읽으면 계산 순서가 된다

RMSE = Root Mean Square Error.
뒤에서 앞으로 읽으면 그대로 계산 순서다.

#### 1. Error (오차)

예측값과 정답의 차이를 구한다.

```
정답 10, 예측 12  →  오차 = 12 - 10 = 2
```

#### 2. Square (제곱)

오차를 제곱한다.

```
2  →  2² = 4
```

제곱하는 이유는 두 가지다.
첫째, 오차가 음수일 수 있는데(예측이 정답보다 작으면 -3 같은 값) 제곱하면 부호가 사라져 전부 양수가 된다. 틀린 정도는 방향과 상관없이 크기만 중요하기 때문이다.
둘째, 제곱하면 큰 오차에 더 큰 벌점이 부여된다.
대신 제곱을 하기 때문에, 아주 큰 오차(outlier)에 민감하다는 특징도 있다.

#### 3. Mean (평균)

제곱한 오차들을 전부 더한 뒤 개수로 나눈다.

#### 4. Root (제곱근)

마지막에 루트(√)를 씌운다.

2에서 제곱하면서 단위가 바뀌었다.
예를 들어 기온 예측이면 "도"가 "도²"이 된다.
루트를 씌우면 원래 단위로 돌아가서, "평균적으로 몇 도 정도 틀렸다"고 해석할 수 있다.

### 전체 흐름

```
정답: [10, 20, 30]
예측: [12, 18, 33]

① Error:   [12-10, 18-20, 33-30] = [2, -2, 3]
② Square:  [4, 4, 9]
③ Mean:    (4 + 4 + 9) / 3 = 5.67
④ Root:    √5.67 ≈ 2.38
```

결과 해석: 이 모델의 예측은 정답에서 평균적으로 약 2.38 정도 벗어나 있다.

절댓값을 사용하는 MAE(Mean Absolute Error)라는 방법도 있다.
MAE는 모든 오차를 비슷하게 다루지만, RMSE는 제곱을 사용하기 때문에 큰 오차에 더 큰 벌점을 준다.
그래서 큰 실수를 특히 더 줄이고 싶을 때 RMSE를 자주 사용한다.

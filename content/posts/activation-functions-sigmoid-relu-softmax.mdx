신경망은 숫자를 입력받아 숫자를 출력한다. 이때 숫자들을 곱하고 더하는 계산을 반복해서 원하는 결과를 만들어낸다.
하지만 곱하고 더하는 계산만 반복하면, 단순한 관계밖에 표현하지 못한다.
그래서 계산 중간에 활성화 함수를 넣는다. 활성화 함수는 계산 결과를 특정 규칙에 따라 반환하는 함수이다.
예를 들어 음수를 전부 0으로 바꾸거나, 입력값을 특정 범위로 변환하는 식이다.
이 변환 덕분에 신경망은 단순한 관계를 넘어 복잡한 관계도 표현할 수 있게 된다.

초기 신경망 모델은 생물학적 뉴런에서 영감을 받아 만들어졌다.
생물학적 뉴런이 신호를 받고 "활성화할지 말지" 결정하는 과정을 수학적으로 모방한 것이고, 그래서 이름이 활성화 함수인 것이라고 한다.

## Sigmoid Activation Function

입력값을 0~1 범위로 변환한다.

### 동작 원리

`1 / (1 + e^(-z))`

- z가 아주 크면 → e^(-z)가 0에 가까워짐 → 결과가 1에 가까움
- z가 아주 작으면(음수) → e^(-z)가 매우 커짐 → 결과가 0에 가까움
- z = 0이면 → e^0 = 1 → 결과가 정확히 0.5

이 S자 곡선이 유용한 이유는 현실의 판정과 비슷하기 때문이다. 시험 점수가 60점 근처면 합격/불합격이 민감하게 갈리지만, 20점이면 확실히 불합격이고 95점이면 확실히 합격이다. 출력을 0과 1로 가를 때는 보통 0.5를 기준으로 사용하지만, 문제에 따라 다른 기준값(threshold)을 쓰기도 한다.

### 실제 사용 방식

원본 입력 x를 먼저 `z = weight * x + bias`로 변환한 다음 `sigmoid(z)`를 한다.
예를 들어 시험 점수 x로 합격 여부를 판정한다면 `z = 0.1 * x - 6` 같은 식이다. weight는 점수 1점 차이가 얼마나 큰 영향을 주는지를 조절하고, bias는 기준점을 어디에 둘지를 정한다.

## ReLU Activation Function

### 정류(rectification)

원래 전기/전자공학 용어다. 양쪽으로 왔다 갔다 하는 교류를 한쪽 방향으로만 흐르게 만드는 것을 정류라고 하고, 이 역할을 하는 장치가 정류기(rectifier)다.
정류기의 핵심 부품인 다이오드는 (+) 방향 전류는 그대로 통과시키고, (-) 방향 전류는 막아버린다. ReLU의 동작 방식이 다이오드의 정류 동작과 비슷하다.

### ReLU란

Rectified Linear Unit(정류 선형 유닛)의 약자다.

- 입력이 0보다 크면 → 그대로 내보냄
- 입력이 0 이하면 → 0을 내보냄

### 시그모이드 대신 ReLU를 쓰는 이유

#### 기울기가 왜 중요한가

신경망의 학습은 틀린 만큼 거꾸로 거슬러 올라가며 조금씩 고친다. 이걸 역전파(backpropagation)라고 한다. 이때 각 부분을 얼마나 고쳐야 하는지 알려주는 게 기울기(gradient)다.

- 기울기가 크면: 많이 고치라는 강한 신호
- 기울기가 0에 가까우면: 학습이 거의 안 일어남

#### 시그모이드의 문제: 양 끝이 평평하다

시그모이드 곡선은 가운데(z=0 근처)는 경사가 있지만, 양 끝(입력이 아주 크거나 작은 곳)은 거의 수평이다. 이 부분에서는 입력을 바꿔도 출력이 거의 안 변한다. 이걸 포화(saturate)됐다고 한다.

```
1.0 |                         ___________
    |                     ___/
    |                   _/
0.5 |· · · · · · · · ·/· · · · · · · · ·
    |                /
    |             __/
0.0 |____________/
    +-----|-----|-----|-----|-----|----> z
         -4    -2     0     2     4
```

#### 소실이 일어나는 이유: 곱셈이 쌓인다

역전파는 출력층에서 입력층 쪽으로 기울기를 계속 곱해가며 전달한다. 시그모이드의 기울기는 최대 0.25이고 절대 1을 넘지 못한다. 1보다 작은 수를 계속 곱하면 급격히 줄어든다.

예: 10개 층을 지나면 0.25^10 ≈ 0.00000095

입력층 근처에 도달할 즈음에는 기울기가 사실상 사라진다. 이게 기울기 소실(vanishing gradient)이다. 출력층 가까운 층들은 그럭저럭 학습되지만, 입력층에 가까운 깊은 층들은 거의 학습이 안 된다.

### ReLU는 왜 괜찮은가

ReLU는 양수 영역에서는 기울기가 1이라, 시그모이드처럼 기울기가 급격히 작아지지 않는다. 1을 아무리 곱해도 1이므로 신호가 줄어들지 않고 깊은 층까지 잘 전달된다. 이 덕분에 깊은 신경망에서도 학습이 잘 되었고, 계산도 단순해서 ReLU가 표준처럼 사용되기 시작했다.

다만 ReLU도 문제가 있다.
입력이 계속 음수로 들어오면 출력과 기울기가 모두 0이 되어 뉴런이 아예 학습되지 않는 경우가 있다. 이를 dead ReLU 문제라고 한다.

## Softmax Activation Function

### 소프트맥스란

시그모이드가 값 하나를 0~1로 만들었다면, 소프트맥스는 여러 값을 동시에 확률들의 집합으로 만든다.

여러 개의 점수(score)를 받아서, 전부 0~1 사이이면서 합이 정확히 1이 되는 확률 분포로 바꿔준다. 예를 들어 강아지/고양이/새 분류에서 점수가 `[2.0, 1.0, 0.1]`로 나오면, 소프트맥스가 `[0.659, 0.242, 0.099]`로 바꿔준다. 합이 1이고, 점수가 제일 높았던 강아지가 제일 높은 확률을 받는다.

### 왜 확률 분포로 만드는가

점수 그대로는 "2.0이 1.0보다 높다" 정도만 알 수 있다. 확률로 바꾸면 "강아지일 가능성이 65.9%"처럼 해석할 수 있고, 모델이 얼마나 확신하는지도 알 수 있다. 특히 다중 분류에서는 softmax 출력과 cross entropy loss를 함께 사용하는 경우가 많다.

### 계산 방식

각 값을 지수(e의 거듭제곱)로 만든 다음, 전체 지수들의 합으로 나눈다. 나눠주기 때문에 자동으로 합이 1이 된다.

```
softmax(z_i) = e^z_i / (e^z_1 + e^z_2 + ... + e^z_n)
```

### 수치 안정성 문제

위 수식을 그대로 코드로 옮기면 문제가 있다. z가 큰 값이면 e^z가 컴퓨터가 표현할 수 있는 범위를 넘어서 오버플로우가 생긴다.
해결법은 모든 점수에서 최댓값을 빼는 것이다. 가장 큰 값이 0이 되어 e^0 = 1이 되고, 나머지는 전부 음수라 e^(음수)는 0~1 사이이므로 큰 양수에 대한 exp 오버플로우를 효과적으로 방지할 수 있다. 분자와 분모에 같은 값을 곱한 셈이라 약분되므로 결과 확률은 똑같다.

### 시그모이드와의 관계

둘 다 점수를 확률로 바꾸는 함수다.

- 시그모이드: 2지선다(예/아니오) → 출력 하나 (예일 확률 p, 아니오는 1-p)
- 소프트맥스: 여러 선택지(강아지/고양이/새 ...) → 선택지 개수만큼 출력 (합이 1)

2지선다는 여러 선택지의 가장 작은 경우다. 실제로 클래스가 2개인 softmax는 sigmoid 형태로 정리할 수 있다. 소프트맥스의 "최댓값 빼기" 트릭도 시그모이드 수식에서 분자·분모를 같은 값으로 나누는 것과 같은 원리다. 둘 다 결과는 안 바뀌면서 계산만 안전해진다.

Adam은 Adaptive Moment Estimation의 줄임말이다.
Gradient Descent는 매번 현재 gradient만 보고 움직인다.
반면 Adam은 gradient의 최근 방향 흐름과 크기 흐름을 기억하면서 parameter를 업데이트하는 optimizer다.

## Gradient는 목표 방향이 아니다.

점 하나의 현재 위치를 current, 목표 위치를 target이라고 하자.

```
loss = (current.x - target.x)² + (current.y - target.y)²

gradient = {
  x: 2 * (current.x - target.x),
  y: 2 * (current.y - target.y),
}
```

gradient는 loss가 커지는 방향이다.
즉 목표로 가는 방향이 아니라, 오차가 더 커지는 방향이다.
그래서 optimizer는 gradient 방향으로 이동하지 않고 gradient를 뺀다.

```
parameter = parameter - 이동량
```

데모에서 빨간선은 gradient 방향이다. 즉 loss가 커지는 방향이다.
반대로 초록선은 실제 이동 방향이다. 그래서 빨간선과 초록선이 반대 방향처럼 보일 수 있다.

## Momentum: 이전 방향을 기억하기

일반 Gradient Descent는 매번 현재 gradient만 본다.
그런데 gradient는 매번 조금씩 흔들릴 수 있다.
현재 gradient 하나만 보면 움직임이 불안정해질 수 있다.

Momentum은 최근 gradient 방향을 기억해서 더 부드럽게 움직이게 한다.
Adam에서는 이 값이 m이다.

```
m = beta1 * 이전 m + (1 - beta1) * 현재 gradient
```

그래서 m은 현재 gradient 하나가 아니라, 최근 gradient들의 평균적인 방향 흐름이다.
데모에서 보라선은 m 방향이다.

- 빨간선: 지금 이 순간의 gradient
- 보라선: 최근 gradient들의 평균 흐름

## Moving Average: 흐름을 계속 업데이트하기

Moving Average는 최근 값들을 반영한 평균이다.

```
새 평균 = 이전 평균 * 비율 + 현재 값 * 비율
```

Adam은 이 아이디어를 두 번 쓴다.

- m: gradient의 이동 평균. 방향의 흐름을 기억
- v: gradient²의 이동 평균. 크기의 흐름을 기억

## Second Moment v: gradient 크기를 기억하기

Adam에서 v는 gradient를 제곱한 값의 이동 평균이다.

```
v = beta2 * 이전 v + (1 - beta2) * gradient²
```

gradient를 제곱하면 방향 정보는 사라지고 크기 정보만 남는다.
즉 v는 이렇게 이해할 수 있다.
"최근에 gradient가 얼마나 크게 튀었는지를 기억하는 값"

나중에 Adam은 이동량을 계산할 때 sqrt(v)로 나눈다.

```
m / sqrt(v)
```

그러면 gradient가 자주 크게 나오는 방향은 이동량이 줄어든다.
즉 너무 크게 튀는 방향에서는 조심해서 움직인다.

## Bias Correction: 초반 보정

Adam에서 m과 v는 보통 0에서 시작한다.
처음에는 문제가 생긴다.
예를 들어 첫 gradient가 10이고 beta1 = 0.9라면

```
m = 0.9 * 0 + 0.1 * 10
m = 1
```

실제 gradient는 10인데 m은 1밖에 안 된다.
왜냐하면 처음 값이 0이었기 때문이다.
그래서 Adam은 초반 값을 보정한다.

```
m_hat = m / (1 - beta1^t)
v_hat = v / (1 - beta2^t)
```

여기서 t는 몇 번째 업데이트인지다.

## Epsilon: 0으로 나누는 것 방지

Adam의 최종 이동량은 대략 이렇게 계산된다.

```
stepDelta = learning_rate * m_hat / (sqrt(v_hat) + epsilon)
```

그리고 parameter는 이 값을 빼서 업데이트한다.

```
parameter = parameter - stepDelta
```

여기서 epsilon은 아주 작은 숫자다.

```
epsilon = 0.00000001
```

v_hat이 너무 작거나 0이면 나눗셈 문제가 생길 수 있다.
그래서 분모에 작은 값을 더한다.

## Adam 전체 흐름

1. gradient를 구한다.
2. m을 업데이트한다: gradient 방향의 이동 평균
3. v를 업데이트한다: gradient 크기 제곱의 이동 평균
4. m_hat, v_hat으로 bias correction을 한다: 초반 0 시작 문제를 보정
5. stepDelta를 계산한다.
6. parameter를 업데이트한다: parameter = parameter - stepDelta

Adam은 gradient의 방향 흐름과 크기 흐름을 기억하면서 parameter를 업데이트하는 optimizer다.
이 데모에서 점들이 ADAM 글자를 찾아가는 과정은, 결국 parameter들이 loss를 줄이는 방향으로 업데이트되는 과정을 시각화한 것이다.

### 코드

```ts
/**
 * 추천 순서:
 * 1. loss = (x - targetX)^2 + (y - targetY)^2 라고 둘 때 gradient를 구한다.
 * 2. m = beta1 * m + (1 - beta1) * gradient
 * 3. v = beta2 * v + (1 - beta2) * gradient^2
 * 4. mHat, vHat으로 bias correction을 적용한다.
 * 5. current - learningRate * mHat / (sqrt(vHat) + epsilon)으로 위치를 업데이트한다.
 *
 * @param input.current 현재 위치. Adam에서 학습할 parameter에 해당한다.
 * @param input.previousTrace 이전 Adam trace (m, v). 이전 step까지 누적된 Adam 내부 상태. m, v는 이전 gradient 흐름과 gradient 크기 흐름을 기억한다.
 * @param input.target 목표 위치. loss = (current.x - target.x)^2 + (current.y - target.y)^2 기준이 된다.
 * @param input.t 몇 번째 업데이트인지 나타내는 step 번호. Adam bias correction에서 beta1^t, beta2^t 계산에 사용한다.
 *
 */
export function applyAdamStep(input: AdamStepInput): AdamStepResult {
  const learningRate = 1.2;
  // 이전 m을 90% 유지, 현재 gradient를 10% 반영
  const beta1 = 0.9;
  // 이전 v를 99.9% 유지, 현재 gradient 제곱을 0.1% 반영
  const beta2 = 0.999;
  const epsilon = 1e-8;
  // MSE를 미분해서 나온 상수
  const mseDerivativeScale = 2;

  // 1. gradient
  const gradient: Vector2 = {
    x: mseDerivativeScale * (input.current.x - input.target.x),
    y: mseDerivativeScale * (input.current.y - input.target.y),
  };

  // 2. first moment: gradient의 이동 평균을 계산
  const m: Vector2 = {
    x: beta1 * input.previousTrace.m.x + (1 - beta1) * gradient.x,
    y: beta1 * input.previousTrace.m.y + (1 - beta1) * gradient.y,
  };

  // 3. second moment: gradient 제곱의 이동 평균을 계산
  const v: Vector2 = {
    x: beta2 * input.previousTrace.v.x + (1 - beta2) * gradient.x ** 2,
    y: beta2 * input.previousTrace.v.y + (1 - beta2) * gradient.y ** 2,
  };

  // 4. bias correction: 초반에 m과 v가 너무 작게 계산되는 문제를 보정
  const mHat: Vector2 = {
    x: m.x / (1 - beta1 ** input.t),
    y: m.y / (1 - beta1 ** input.t),
  };
  const vHat: Vector2 = {
    x: v.x / (1 - beta2 ** input.t),
    y: v.y / (1 - beta2 ** input.t),
  };

  // 5. update: 앞에서 구한 mHat, vHat을 이용해서 실제로 parameter를 얼마나 움직일지 정하고 업데이트

  // 이번 step에서 빼 줄 변화량
  const stepDelta: Vector2 = {
    x: learningRate * mHat.x / (Math.sqrt(vHat.x) + epsilon),
    y: learningRate * mHat.y / (Math.sqrt(vHat.y) + epsilon),
  };
  // current - stepDelta
  const nextPosition: Vector2 = {
    x: input.current.x - stepDelta.x,
    y: input.current.y - stepDelta.y,
  };

  return {
    nextPosition,
    nextTrace: {
      m,
      v,
      stepDelta,
      gradient,
      mHat,
      vHat,
    },
  };
}
```

### 실행

[데모 링크](https://doing-something.github.io/playground/adam-optimizer-points/)

각 점은

1. 현재 점 위치를 본다.
2. target과 얼마나 떨어졌는지 계산한다.
3. gradient를 구한다.
4. m으로 최근 방향 흐름을 기억한다.
5. v로 최근 크기 흐름을 기억한다.
6. 보정한 뒤 실제 이동 방향을 정한다.
7. 점이 target 쪽으로 이동한다.

이 문제는 단순한 거리 줄이기 문제라 Gradient Descent로도 잘 풀린다.
그래서 Adam과 Gradient Descent의 차이는 이 데모에서 눈으로 크게 드러나지 않는다.
이 데모는 성능 비교가 아니라 Adam 내부 계산 흐름을 보기 위한 것이다.

목표는 데이터를 보고 weights를 조정해서 예측을 점점 정확하게 만드는 것.

## 1. Linear Prediction - 지금 weights로 예측해보기

```
y_pred = X @ weights
```

X는 입력 데이터, weights는 각 입력의 중요도이다.  
@는 행렬 곱셈으로, 입력마다 가중치를 곱해서 더하는 연산을 한꺼번에 처리한다.

```
X = [[2, 3],   weights = [[0.5],
   [4, 1]]               [0.2]]

y_pred = [[2×0.5 + 3×0.2],   = [[1.6],
		 [4×0.5 + 1×0.2]]      [2.2]]
```

처음에는 weights를 아무 숫자로 시작하므로 예측이 틀리는 게 당연하다.

## 2. MSE Loss - 얼마나 틀렸는지 숫자 하나로

정답(y_true)이 있을 때, 예측값이 얼마나 벗어났는지 측정한다.

```
MSE = 평균((y_pred - y_true)²)
```

제곱을 하는 이유는 두 가지다.
- 예측이 위로 빗나가든 아래로 빗나가든 둘 다 틀린 것으로 취급한다. (음수 제거)
- 많이 틀릴수록 훨씬 크게 별점을 준다. (오차 2 -> 벌점 4, 오차 4 -> 벌점 16)

```
y_pred = [1.6, 2.2]
y_true = [2.0, 2.0]

오차: [-0.4, 0.2]
오차²: [0.16, 0.04]
MSE = (0.16 + 0.04) / 2 = 0.10
```

이 숫자가 낮을수록 모델이 잘 맞히고 있다.

## 3. Gradient - 어느 방향으로 weights를 바꿔야 loss가 줄어드나.

Gradient는 각 weight를 어느 방향으로 바꿔야 loss가 줄어드는지 알려주는 값이다.

수식
```
gradient = (2/n) × Xᵀ @ (y_pred - y_true)
```

이건 MSE Loss를 weights에 대해 미분한 결과이다.

```
y_pred = X @ weights
error = y_pred - y_true
MSE = (1/n) * sum(error^2)

gradient = ∂MSE / ∂weights # weights에 대한 MSE의 변화량
         = (2/n) * Xᵀ @ (y_pred - y_true)
```

### Xᵀ: 전치

```
// 입력 데이터 X
X =
[[1, 2],
 [1, 3]]
 
// 전치하면
Xᵀ =
[[1, 1],
 [2, 3]]
```

원래 X는 이렇다.
- 행: 데이터
- 열: feature (모델이 예측할 때 참고하는 입력 값)

우리가 구하고 싶은 것은 feature별 gradient이다.  
그래서 데이터를 기준으로 모여 있는 X를 feature 기준으로 재배치해야 한다.

### 왜 오차와 곱하나?

각 feature가 현재 오차에 얼마나 책임이 있는지 계산하는 과정으로, 얼마나 틀렸는지를 반영하기 위해서다.
(많이 틀린 것은 크게 반영, 조금 틀린 것은 작게 반영)

예시
```
오차 = y_pred - y_true = [-0.4, 0.2]

gradient = (2/2) × Xᵀ @ [-0.4, 0.2]
	     = X를 전치한 뒤 오차와 곱하는 것
```

오차는 방향과 크기를 알려준다.  
오차가 음수이면 예측이 실제보다 낮다는 뜻이고,  
오차가 양수이면 예측이 실제보다 높다는 뜻이다.  
또 오차의 절댓값이 클수록 많이 틀렸다는 뜻이다.  

여기에 feature 값을 곱하면,  
그 feature가 큰 값으로 예측에 많이 관여한 경우 더 큰 gradient가 나오고,   
작게 관여한 경우 더 작은 gradient가 나온다.  
그래서 feature * error를 계산하면 각 weight를 어느 방향으로 얼마나 조정해야 하는지 알 수 있다.  

## 4. Gradient Descent — gradient 반대 방향으로 weights 조정

```
weights = weights - learning_rate × gradient
```

gradient의 반대 방향으로 weights를 조금씩 움직인다.  
gradient는 현재 위치에서 loss가 가장 빨리 증가하는 방향을 알려준다.   
따라서 loss를 줄이려면 그 반대로 가야한다.  

```
weights = [[0.5],   gradient = [[0.3],   learning_rate = 0.1
		  [0.2]]               [-0.1]]

새 weights = [[0.5 - 0.1×0.3],   = [[0.47],
			 [0.2 - 0.1×(-0.1]]     [0.21]]
```

이 과정을 반복하면 weights가 점점 loss를 낮추는 방향으로 이동한다.

## 5. Learning Rate — 한 번에 얼마나 크게 움직일지

gradient 방향으로 weight를 얼마나 크게 움직일지 정하는 값이다.

```
learning_rate = 0.1      // 적당히 빠르게 수렴
learning_rate = 10       // 발산할 위험 (loss가 오히려 커짐)
learning_rate = 0.00001  // 너무 느림
```

보통 0.001 ~ 0.1 정도에서 시작해 보고 조정한다.

## 6. Epoch — 전체 데이터를 한 번 다 보는 것

데이터 전체를 한 번 모두 사용한 것이 1 epoch다. (epoch 안에서 weight 업데이트는 여러 번 일어날 수 있다.)
보통 수십~수백 epoch를 돌린다.

```
Epoch 1: loss = 2.4
Epoch 5: loss = 1.1
Epoch 20: loss = 0.3
Epoch 50: loss = 0.05  // 점점 줄어든다
```

## 전체 흐름 한눈에

```
weights 초기화
  ↓
[Epoch 시작]
  ↓
y_pred = X @ weights               // 예측
  ↓
loss = MSE(y_pred, y_true)         // 얼마나 틀렸나
  ↓
gradient 계산                       // 어느 방향으로 조정해야 하나
  ↓
weights = weights - lr × gradient  // 조정
  ↓
[다음 Epoch 반복]
```

weights가 처음에는 아무 의미 없는 숫자지만, 이 루프를 반복하면서 데이터에 맞는 값으로 조금씩 찾아간다.
